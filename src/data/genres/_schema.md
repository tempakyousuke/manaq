# ジャンルJSONの形式

`src/data/genres/` に `*.json` を追加すると、自動的に新しいジャンルとしてサイトに表示されます。
（`_` で始まるファイルはこのドキュメント用で、`*.json` 以外は読み込まれません）

```jsonc
{
  "id": "claude-code",          // URLに使う一意なID（省略時はファイル名）
  "title": "Claude Code",       // 表示名
  "description": "一行説明",
  "icon": "🤖",                  // 絵文字アイコン
  "order": 1,                    // 並び順（小さいほど先頭。省略可）
  "updated": "2026-06-23",       // 最終更新日のデフォルト（任意。topic/question で上書き可）
  "topics": [
    {
      "id": "basics",           // ジャンル内で一意
      "title": "基本概念",
      "summary": "このトピックの概要（任意）",
      "updated": "2026-06-23",  // このトピックの設問が継承する更新日（任意）
      "lessons": [
        { "heading": "見出し", "body": "本文。\n\nで段落、`code` でインラインコード。" }
      ],
      "questions": [
        {
          "question": "問題文",
          "choices": ["選択肢A", "選択肢B", "選択肢C", "選択肢D"],
          "answer": 0,          // 正解の choices インデックス（0始まり）
          "explanation": "解説文",
          "updated": "2026-06-23" // この設問だけの更新日（任意。topic/genre より優先）
        }
      ]
    }
  ]
}
```

## 最終更新日（`updated`）の継承

`updated` は陳腐化しやすい設問の鮮度を示すフィールドです。設問ごとに書けますが、
未指定なら **question → topic → genre** の順でフォールバックします。
ジャンルに1つ書けば全設問に反映され、古くなった設問だけ個別に上書きできます。
クイズで解答するとその設問の解説欄に表示されます。形式は `"YYYY-MM-DD"` か `"YYYY-MM"`。

## 新しいジャンルを追加する手順

1. このフォルダに `<ジャンル名>.json` を作成する
2. 上の形式に沿って `topics` を書く
3. 開発サーバーが動いていれば自動で反映される
